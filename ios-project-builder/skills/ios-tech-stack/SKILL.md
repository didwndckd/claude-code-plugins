# 기술 스택 선택 스킬

`docs/specs/base.md`의 요구사항을 바탕으로 기술 스택 후보를 제안하고, user 선택을 받아 `CLAUDE.md`의 `## 기술 스택` 섹션을 작성/갱신한다.

## 산출물

- 경로: `CLAUDE.md` (프로젝트 루트)
- 파일이 없으면 새로 만들고 `## 기술 스택` 섹션만 추가한다. 이미 있으면 동일 헤더 섹션만 추가/갱신하고 다른 내용은 보존한다.
- 템플릿:

```markdown
## 기술 스택

| 카테고리 | 선택 |
| --- | --- |
| UI | <선택> |
| 반응형 | <선택> |
| 비동기 | <선택> |
| 네트워크 | <선택> |
```

표의 4개 행은 **기본 행**이며, 아래 "추가 카테고리" 조건에 해당할 때만 행을 추가할 수 있다.

## 사전 조건

- `docs/specs/base.md`가 존재해야 한다. 없으면 멈추고 user에게 `spec-analysis` 스킬을 먼저 돌리라고 안내.

## 작업 절차

### 1. 입력 수집

- `docs/specs/base.md`를 Read하여 "1. 요구 기술 스택" 섹션과 "user 결정 필요 항목" 중 기술 스택 관련 항목을 컨텍스트로 보관.
- `extra_notes`가 있으면 결정 가중치로 사용.
- `CLAUDE.md`가 이미 있으면 Read하여 다른 섹션 보존용으로 보관.

### 2. 결정

base.md "1. 요구 기술 스택"에 **명시된 요구**는 다시 묻지 않고 그대로 확정. **자유/명시되지 않음**인 항목만 user 선택을 받는다.

기본 카테고리(항상 포함)와 대표 후보:

- **UI**: UIKit / SwiftUI / 혼합
- **반응형**: RxSwift / Combine / 사용 안 함
- **비동기**: Swift Concurrency / RxSwift / Combine / 혼합
- **네트워크**: URLSession / Alamofire / Moya

위 후보는 **권장 기본값**이며, 아래 조건을 만족할 때 후보를 추가할 수 있다:

- base.md "1. 요구 기술 스택"이나 "user 결정 필요 항목"에 해당 라이브러리가 등장
- `extra_notes`에서 명시적으로 언급
- base.md "기능 목록"의 요구사항에서 명확히 도출 (예: GraphQL 백엔드 → Apollo, 차트 화면 → Charts/DGCharts)

위 조건 없이 학습 지식에서 떠올린 라이브러리(Kingfisher, Swinject 등)를 검증 없이 후보로 던지지 않는다.

추가 카테고리는 아래 조건 중 하나라도 만족할 때만 표에 행을 추가:

- base.md "1. 요구 기술 스택" 섹션에 해당 카테고리/라이브러리가 등장
- `extra_notes`에서 명시적으로 언급
- base.md "기능 목록"이 해당 카테고리 없이는 구현 불가능 (예: 오프라인 캐시 기능 → 영속성, 이미지 다수 표시 → 이미지 캐시)

추가 카테고리의 후보 역시 위 추가 조건을 따른다 (대표 후보를 제시한 뒤 맥락에 맞춰 확장 가능).

선택 도구는 `AskUserQuestion`. 기본은 자유 항목 전부를 한 번의 호출에 묶어서 묻기. 도구 제약으로 분할이 필요한 경우에만 분할 호출.

### 3. CLAUDE.md 작성/갱신

산출물 섹션의 템플릿대로 `## 기술 스택` 섹션을 작성한다. 템플릿 외 항목은 추가하지 않는다.

기존 `CLAUDE.md`가 있으면 동일 헤더(`## 기술 스택`)만 교체하고 다른 섹션/내용은 그대로 보존한다.

## 제약

- **후보 추가는 맥락 기반**: 대표 후보 외 라이브러리는 base.md, `extra_notes`, 또는 base.md 기능 요구에서 도출 가능할 때만 후보로 제시. 학습 지식에서 임의로 떠올린 라이브러리를 검증 없이 던지지 않는다.
- **메타데이터 추가 금지**: 결정일, 근거 문서, 결정 근거 본문, 트레이드오프 설명 등을 출력에 넣지 않는다. 표는 카테고리·선택 값만 담는다.
- 카테고리 행 추가는 위 "추가 카테고리" 조건에 해당할 때만 허용. 그 외 임의 추가 금지.
- `CLAUDE.md`의 `## 기술 스택` 외 섹션은 절대 임의 변경하지 않는다.
- base.md에 명시된 요구는 user에게 다시 묻지 않고 확정 처리한다 (변경하려면 base.md를 먼저 갱신).
- 아키텍처/디자인 패턴 결정은 이 스킬 범위 밖.
- 코드 작성, 폴더 구조 결정, 프로젝트 생성은 이 스킬 범위 밖.

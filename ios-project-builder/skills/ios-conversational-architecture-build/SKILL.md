# 대화형 아키텍처 빌더 (iOS)

## 기본 자세

이 스킬은 정해진 아키텍처 틀을 가정하지 않는다. user가 원하는 형태로 자유롭게 설계하도록 돕는 것이 목적이다.

iOS 전문가로서 user의 발화에 따라 응답한다. 정해진 질문 시퀀스, 정해진 결정 축, 정해진 후보 리스트는 없다 — 모두 user가 요청할 때만 제공.

레이어 분할(Presentation/Domain/Data 식), 의존 방향, 폴더링 규칙 같은 항목도 기본 가정으로 두지 않는다. user가 그런 구성을 원하면 따라가고, 다른 구성(피처 기반, 단일 레이어, MV/MVVM/MVC만, 커스텀 등)을 원하면 그대로 따라간다.

## 작업 절차

### 1. 대화형 설계

user 주도로 아키텍처 구조를 설계한다. 모델은 user의 발화를 받아 정리하고, 충돌이나 빠진 부분이 보일 때만 짧게 묻는다. 결정 축을 미리 펼쳐놓지 않는다.

user가 "어떤 게 좋을까", "추천해줘", "장단점은?" 같이 명시적으로 의견을 요청할 때만 전문가 의견을 낸다. 그 외엔 user가 말한 구성을 그대로 받아 적는다.

각 결정은 임시 메모로 누적한다. user가 "정리해줘" 또는 "확정 가자"라고 하면 다음 단계로.

### 2. 종합 확정

누적된 결정 전체를 한 번에 user에게 보여주고 `AskUserQuestion`으로 단일 확정 질문. "확정" → 다음 단계. "수정" → 1단계로 복귀하여 해당 항목만 다시 논의.

### 3. 폴더 생성

확정된 구조의 폴더 트리를 생성한다.

- 생성 루트는 `AskUserQuestion`으로 user에게 묻는다. 기본 추천은 프로젝트 루트(`.`).
- 빈 폴더 유지 수단(`.gitkeep` 등)이 필요한지 user에게 묻고 처리.
- 각 폴더는 `mkdir -p`로 멱등하게 생성.

### 4. CLAUDE.md `## 아키텍처 구조` 섹션 작성

`CLAUDE.md`(프로젝트 루트)에 `## 아키텍처 구조` 섹션을 작성/갱신한다. 파일이 없으면 새로 만들고, 있으면 동일 헤더 섹션만 교체/추가하고 다른 섹션은 보존.

섹션 본문은 user가 확정한 결정만 반영한다. 항목 구성은 case마다 다르다 — 결정되지 않은 항목은 적지 않는다. 일반적으로 포함될 수 있는 것:

- 채택된 아키텍처/패턴 한 줄 요약 (user가 명명을 정한 경우).
- 확정된 폴더 트리 (3단계에서 만든 것과 동일).
- user가 합의한 그 외 결정 (의존 방향, DI 위치, 모듈 경계, 명명 규칙 등 — 결정된 것만).

메타데이터(결정일, 근거, 트레이드오프 설명) 추가 금지.

### 5. 단위별 룰 작성

확정된 구조 단위 각각에 대해 `.claude/rules/<단위명>.md`를 작성한다. "단위"는 user가 정한 분할 기준에 따라 다르다 — 레이어, 피처, 모듈, 또는 단일 단위일 수도 있다. 분할 자체가 없으면 룰 파일도 단일.

- 룰 파일은 해당 단위에서 코드 작성 시 따라야 할 규칙을 담는다 (책임, 의존 방향, 명명, 금지 사항 등 — user 결정에서 도출되는 것만).
- 내용은 user와 합의한 결정에서 도출하며, 모호한 부분은 user에게 다시 묻는다.
- 기존 파일이 있으면 user에게 덮어쓸지 묻는다.

```
mkdir -p .claude/rules
```

### 6. 후속 빌더 호출 (선택)

user 결정에 네트워크 베이스 코드 생성이 포함되면 `ios-network-base-build`를 호출한다.

인자:
- `path`: 3단계에서 결정된 네트워크 베이스 위치.
- `network`: 네트워크 라이브러리. 인자가 비면 user에게 묻는다.
- `async`: 비동기 패턴. 인자가 비면 user에게 묻는다.

그 외 별도 빌더 스킬이 필요한 경우 동일하게 `Skill` 도구로 호출. 인자가 부족하면 호출 전에 user에게 묻는다.

## 제약

- 대화는 iOS 전문가 관점에서 진행. user 발화에 근거가 없는 추측으로 결정을 채우지 않는다. 추측이 불가피하면 그렇게 명시한다.
- 후보 리스트를 표면에 강제로 노출하지 않는다. 추천/위임은 user가 명시적으로 요청할 때만.
- `CLAUDE.md`의 `## 아키텍처 구조` 외 섹션은 임의 변경하지 않는다.
- 폴더 생성, 룰 작성, 후속 빌더 호출은 종합 확정 이후에만 실행.
- 코드 작성, 도메인별 구현, 프로젝트 파일(.xcodeproj) 변경은 이 스킬 범위 밖.
